# Careme Android TWA

This project is generated by Bubblewrap from `twa-manifest.json`.

The PWA manifest at `../internal/static/manifest.webmanifest` cannot declare Android runtime permissions. Bubblewrap reads it for install metadata such as name, colors, icons, shortcuts, protocol handlers, file handlers, and launch handling, but Android `uses-permission` entries are generated from Bubblewrap's Android feature model.

Location delegation is enabled so browser `navigator.geolocation` permission requests use the native Android permission prompt. Bubblewrap 1.24.1 generates the delegation service and permission request activity, but it does not declare an Android location permission for the app. Careme only converts the position to a ZIP code with `enableHighAccuracy: false`, so the generated manifest needs coarse foreground location:

```xml
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
```

After running Bubblewrap regeneration or update commands, run:

```sh
./apply-bubblewrap-patches.sh
```
